# 10-desire-map.md: 욕망 맵

> Paperclip Company OS v0.7
> 표면적 요청 → 실제 문제 → 솔루션 연결
> 기준일: 2026-07-02

## 1. 표면적 요청 (Surface Request)

```
"AI 미디어커머스 운영 OS가 필요하다"
```

사용자가 말한 원래 요청:
- "AI 글쓰기 도구를 만들어 달라"
- "블로그 자동 발행 기능이 있으면 좋겠다"
- "네이버 쇼핑커넥트와 연동해서 수익화하고 싶다"
- "매일 발행하는데 자동으로 돼면"

## 2. 다시 정리한 한 줄 문제 (Real Problem)

```
혼자서 매일 수익형 콘텐츠를 기획 → 제작 → 검수 → 게시 → 회고까지
모두 돌리려면 회사 수준의 운영 구조가 필요한데,
그걸 사람 손으로 다 하면 지속이 안 된다.
```

**이 문제에 도달한 이유:**

### 문제 분석 (왜 "글쓰기 도구"로는 부족한가)

```
사용자가 하던 일:

1. 아침 (10분)
   "뭘 쓸까?" 고민
   ↓
2. 오전 (30분)
   블로그 글 작성 (제목, 구조, 본문)
   ↓
3. 낮 (15분)
   쇼핑커넥트 링크 찾고 배치
   ↓
4. 오후 (20분)
   SNS 올리기 (Instagram, Threads, X)
   ↓
5. 저녁 (10분)
   성과 기록, 다음 전략 고민
   ↓
6. 이틀에 한 번
   "다음엔 뭘 하지?" 전략 재수립

결과: 지속 불가능 (월 2~3글만)

"글쓰기 도구"로 해결 안 되는 부분:

1. 기획: "뭘 쓸까" 고민이 자동화되나? NO
   → AI 글쓰기 도구는 "이미 결정된 주제"만 쓸 수 있음
   → 주제 선택 자체가 가장 어려운 부분

2. 전략: "홈피드 제목" vs "검색 제목" 두 개를 어떻게 쓸 것인가?
   → 같은 글 2개 버전? 아니면?
   → AI 도구는 순수 글만 씀

3. 수익화: 쇼핑커넥트 링크를 어디에 어떻게 배치할 것인가?
   → "링크 여기 추가" 수준이 아니라, "CTA 전략"이 필요
   → 글 완성 후 별도 작업

4. 검수: 대가성 문구는 있나? 가격 기준일은 있나?
   → AI가 실수하는 부분
   → 사용자가 수동으로 다시 확인

5. 출판: 블로그 / 클립 / SNS 각각 다른 형식
   → 글을 복사해서 SNS에 붙일 수 없음
   → 채널별 "변환"이 필요

6. 학습: "지난번 잘 된 제목은?" → 패턴이 자동으로 반영되나?
   → NO, 사용자가 기억했다가 "다음엔 이렇게 쓸게"라고 해야 함

→ 결론: "글쓰기"만으로는 월 1글도 어렵다.
          필요한 건 "회사처럼 돌아가는 시스템"이다.
```

## 3. 다시 정리한 한 줄 (JTBD 형태로, 라벨 없이)

```
"매일 아침 30분,
 오늘 만들 수익형 콘텐츠 1개를 고르고,
 그걸 패키지로 받으면 (블로그 원문 + 썸네일 + 쇼핑 동선까지),
 남는 시간에 게시만 하고
 수익을 쌓을 수 있었으면 좋겠다."
```

**이게 가능하려면:**

```
1. 매일 기회가 자동으로 떠야 하고 (Hermes)
2. 떠난 기회가 이미 점수가 매겨져 있어야 하고 (HomeFeed + Search + Revenue 점수)
3. 선택만 하면 모든 채널이 자동으로 만들어지고 (Content Engine + Profiles)
4. 검수가 자동으로 되어 (Compliance Gate)
5. 게시만 수동으로 하면 (Export + Manual Publish)
6. 다음 주제에 반영되는 (Company Memory)

→ "1인 회사"의 구조
```

## 4. Must 기능과의 연결

### 왜 이 기능들이 필수인가?

```
Must 기능 (13개) ↔ 한 줄 문제의 해결 경로

1. Paperclip HQ 메인 ←→ "매일 아침 30분 시작점"
   (경영 레이어가 없으면 흐름 자체가 없음)

2. Hermes Opportunity Memo ←→ "기회가 자동으로 떠야 함"
   (사용자가 "뭘 쓸까" 고민 시간 제로화)

3. 주제 점수화 ←→ "떠난 기회 평가"
   (홈피드/검색/수익/리스크 4축으로 실시간 판단)

4. 홈판형 제목 생성 ←→ "네이버 피드에서 눌릴 제목"
   (피드 최적화 없으면 조회수 10% 수준)

5. 검색형 구조 생성 ←→ "동일 글로 검색도 잡기"
   (별도 글 작성 X, 1개 글이 2가지 채널 커버)

6. 쇼핑 상품/링크 관리 ←→ "URL 붙여넣기 수준의 입력"
   (이게 복잡하면 수익화 포기. 간단해야 지속됨)

7. 블로그 원문 생성 ←→ "사실상 Export할 완성본"
   (사용자가 "한 번 더 쓰기"를 안 해야 함)

8. 클립 대본 생성 ←→ "재활용으로 추가 조회" (Phase 2)
   (같은 내용 여러 채널로 수익 극대화)

9. SNS 변환 ←→ "채널별 최적화" (Phase 2)
   (복사 붙여넣기 X, 각 플랫폼 특성에 맞게)

10. Compliance Gate ←→ "발행 전 최종 검수"
    (대가성 문구 누락 시 계정 정지 위험)

11. Export Bundle ←→ "게시용 4가지 형식"
    (markdown, html, copy, sns용)

12. Performance Logger ←→ "숫자 3~4개만 입력"
    (성과 기록이 번거우면 단절됨)

13. Company Memory ←→ "다음 주제 선택에 자동 반영"
    (학습 없으면 매달 같은 실수 반복)

→ 13개 모두가 필수인 이유:
   하나라도 빠지면 "시스템"이 아니라 "도구"가 됨
```

## 5. Won't 리스트와 그 이유

### 왜 "이것들은 MVP에 안 넣나?"

| Won't 기능 | 해당 문제 | 왜 제외했나 |
|-----------|----------|-----------|
| **블로그 자동 발행** | "게시까지 자동으로" | 네이버 API 종료 (2020) + 정책 리스크 (낚시글 자동 발행 방지) |
| **SNS 자동 게시** | "모든 채널 자동" | OAuth 복잡도 + 정책 변경 빈번 (Meta, X) + 초기 수익 효과 낮음 |
| **쇼핑커넥트 자동 스크래핑** | "상품 자동으로 찾기" | 네이버 로그인 필요 + 약관 위반 위험 |
| **완전 자동 발행** | "정말 아무것도 안 해도 되게" | 숨김, 저품질 콘텐츠, 광고글 피로도 리스크 |
| **멀티유저 SaaS** | "다른 사람들도 쓸 수 있게" | MVP 목표 = 1인 회사 증명. 확장은 Phase 5+에서 |
| **결제/구독 시스템** | "수익화하기" | SaaS가 아니므로 불필요. 사용자는 쇼핑커넥트 수익만 취함 |
| **영상 자동 생성** | "진짜 완전 자동으로" | 기술 복잡도 높음 + 비용 (video AI) + 클립은 대본까지만 (영상 직접 촬영) |
| **만세력 엔진** | "길일 고르기 같은 부가 기능" | 미디어커머스 핵심과 분리. 우선순위 낮음 |

**공통 원칙:**
```
제외 이유 1: 기술 / 정책 리스크 (자동 발행, 스크래핑)
제외 이유 2: 사용자 체험 리스크 (우리가 "완전 자동"으로 망칠 수 있음)
제외 이유 3: 초기 수익 대비 복잡도 (SNS, 멀티유저)
제외 이유 4: 핵심 문제와 무관 (영상 생성, 만세력)

→ MVP는 "정말 필요한 것"만 + "사용자가 손으로 할 수 밖에 없는 부분" (수동 게시)
```

## 6. 해결 검증 방법

### "이 솔루션이 정말 문제를 푸는가?"

```
가설: "30분 시스템이 있으면 월 4글, ₩500k 달성 가능할까?"

Month 1 검증:
- 사용자가 아침 30분 + 저녁 15분 (총 45분) 투자 가능한가?
- Paperclip이 정말 "고민 없이 선택"할 수 있게 하는가?
- 콘텐츠 품질이 (자동 생성에도) 충분한가?

Month 2 검증:
- 월 2글 이상 지속 가능한가?
- 조회수가 100 > 200 > 250으로 증가하는가?
- Company Memory가 다음 주제에 반영되는가?

Month 3 검증:
- 월 4글 달성했는가?
- 수익이 ₩500k에 가까운가?
- 사용자가 "이 시스템이 없으면 못 한다"고 느끼는가?

→ 위 3가지 모두 "YES"면 문제 해결 증명
```

## 7. 실패 시나리오 (회피해야 할 것)

### 이렇게 되면 안 된다

```
시나리오 1: "구조는 좋은데 시간이 여전히 많이 걸려"
  증상: 사용자가 "아침 1시간, 저녁 1시간" 투자 필요
  원인: 입력 필드 너무 많음 (상품 5개 세부정보 입력 등)
  해결: "URL 붙여넣기"까지 최소화
  
시나리오 2: "품질이 떨어져서 쓸 수 없어"
  증상: 콘텐츠 패키지가 나와도 사용자가 "다시 써야 함"
  원인: AI 모델이 부족하거나 프롬프트가 부정확
  해결: Claude / GPT-4 수준 필수. 테스트 후 출시
  
시나리오 3: "지난번 패턴이 다음엔 안 반영되더라"
  증상: Company Memory가 계속 실패
  원인: 학습 로직이 너무 복잡하거나 버그
  해결: 간단한 규칙부터 시작 (후킹 유형별 조회수 평균만)
  
시나리오 4: "매번 수동 정정이 필요해"
  증상: Compliance Gate가 계속 실패 뜸
  원인: 규칙이 너무 까다롭거나 잘못 설정됨
  해결: High Risk 0건, Medium은 사용자 수동 확인만
  
시나리오 5: "결국 SNS 자동 발행을 원하는데..."
  증상: 사용자가 "게시도 자동으로 해줘"라고 함
  원인: 자동 발행 제약 설명이 부족
  해결: "정책 리스크" 명확히 설명. Phase 2+에서 검토
```

## 8. 제품 vs 솔루션

### "이건 제품인가, 솔루션인가?"

```
제품 (Product):
  = "블로그 글쓰기 AI도구"
  사용자 입장: "편한 도구네" (하지만 혼자여야 함)

솔루션 (Solution):
  = "1인 회사 운영 OS"
  사용자 입장: "내가 경영실장, 결정권자가 됐다"

Paperclip은?
  → "솔루션"이다.
  
이유:
1. AI 도구가 아니라, "시스템 구조"를 제공
2. 사용자는 "쓰는 사람"이 아니라 "결정하는 경영자"
3. 성공의 핵심은 "좋은 글"이 아니라 "좋은 의사결정"
4. 가능성: 사용자가 "다른 채널" (쿠팡, 11번가) 추가 가능

→ 이를 위해:
   - 채널별 Agent 분리 X (Profile만 바꿈)
   - 모든 구조를 "의사결정 중심"으로 설계
   - 승인은 항상 "사용자 오너"가 함
   - AI는 "제안만" 함
```

## 9. 최종 가설 (Theory of Change)

```
우리가 믿는 것:

1. 문제: "혼자는 지속 불가능"
   증거: 사용자가 월 2~3글에서 6개월 정체됨

2. 원인: "회사처럼 분업할 수 없음"
   증거: 기획/제작/검수/출판/분석을 모두 1인이 함

3. 솔루션: "AI + 구조 = 시스템"
   증거: Hermes(조사) + Paperclip(결정) + Agents(실행) + User(승인)

4. 예상 효과: "30분 투자로 월 4글, ₩500k"
   검증: Month 3까지 데이터 수집 후 판단

5. 장기 가능성: "1인이 "회사"처럼 운영"
   비전: 다른 채널 추가 / SaaS화 / 팀 확대

→ 이 가설이 맞다면, Paperclip은 단순 도구가 아니라 "패러다임 전환"이다.
```
