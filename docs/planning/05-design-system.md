## 핵심 설계 원칙

### "하루 30분, 입력 최소화"

모든 UI 설계의 1순위는 **사용자가 30분 안에 완료할 수 있도록 입력 필드를 최소화**하는 것입니다.

```
입력 최소화의 예:
❌ 상품 등록: 상품명, URL, 가격, 수수료, 주의사항, 카테고리, 태그, 설명 (8개 필드)
✓  상품 등록: URL 붙여넣기 (자동 크롤링) / 확인일 입력 (1개 필드)

❌ 성과 기록: 조회수, 클릭, 좋아요, 댓글, 공유, 저장, 직접 수익, 간접 수익 (8개)
✓  성과 기록: 조회수, 클릭 (2개 필드) + 수익은 선택

❌ 주제 선택: 상세 분석 + 카테고리 선택 + 키워드 입력 + 선호도 설정
✓  주제 선택: [선택] 버튼 1개 (Hermes가 모든 분석 완료)
```

## 4. 반응형 설계

### Breakpoints

```css
Mobile:        0px - 640px   (기본)
Tablet:        641px - 1024px
Desktop:       1025px+

메인 컨텐츠 너비:
Mobile:   100% (패딩 16px)
Tablet:   90% (최대 900px)
Desktop:  95% (최대 1400px)

터치 타깃 크기 (D4):
Mobile:   최소 44px (WCAG 2.5.8 AAA 준수)
Desktop:  28px (WCAG 2.5.8 AA 준수)
- 예외: [폐기] 등 파괴적 액션은 44px 유지 + 확인 다이얼로그 추가
```

### 모바일 우선

```
Mobile (아침에 공원에서 스마트폰으로 보기):
- 사이드바: 축소 또는 드롭다운
- 칸반: 가로 스크롤
- 카드: 100% 너비
- 입력: 터치하기 쉽게 최소 44px 높이

Tablet:
- 사이드바: 고정 표시 (축소)
- 메인: 2단 레이아웃 가능
- 우측 패널: 스택형 또는 축소

Desktop:
- 3단 레이아웃 (사이드바 + 메인 + 우측)
- 모든 요소 가시화
- 우측 패널 풀 크기
```

## 6. 접근성 (A11y)

```
WCAG 2.1 AA 준수:

컬러 대비:
  - 텍스트 vs 배경: 최소 4.5:1 (일반), 3:1 (큰 텍스트)
  - 예: #1F2937 텍스트 on #FFFFFF = 14.5:1 ✓

포커스 표시:
  - 버튼 포커스: 파란 테두리 2px + 4px offset
  - 입력 포커스: 파란 테두리 2px

키보드 네비게이션:
  - Tab 순서: 좌상→우하
  - 모든 상호작용 요소 포함

라벨:
  - 모든 입력에 명시적 <label> 태그
  - 아이콘 버튼에 aria-label

이미지 alt:
  - 점수 숫자: 스크린 리더용 "홈피드 점수 72점"
  - 상태 아이콘: "검수 통과", "검수 실패"
```

### 토스트 심각도별 (D1)

**정보성 토스트** (자동 소멸)
```css
자동 닫기: 4초 후 페이드아웃
배경: #DBEAFE (옅은 파랑)
아이콘: ℹ️ (정보)
예: "변경사항이 저장되었습니다"
```

**에러 / 컴플라이언스 토스트** (수동 해제)
```css
자동 소멸: 금지 — 수동으로 [×] 해제 필수
배경: #FEE2E2 (옅은 빨강) / #FFFBEB (옅은 주황)
아이콘: ✗ (실패) / ⚠️ (경고)
재확인 경로: [다시 보기] 또는 [상세 보기] 버튼 표시
예: "검수 실패: 대가성 문구 필수" → [수정하기] 제공
```
