# 08-business-model.md: 비즈니스 모델

> Paperclip Company OS v0.7
> "1인 AI 미디어커머스 회사" 수익 구조
> 기준일: 2026-07-02

---

## 핵심 원칙

**Paperclip Company OS는 SaaS가 아니다.**
- 구독료 없음
- 결제 시스템 없음
- 1인 개인용 운영 시스템
- 사용자가 직접 네이버 쇼핑커넥트 수익화

---

## 1. 수익 흐름

```
사용자가 Paperclip으로 콘텐츠 생성
    ↓
블로그 + 쇼핑커넥트 링크 게시
    ↓
독자가 링크 클릭
    ↓
네이버 쇼핑에서 상품 구매
    ↓
사용자가 쇼핑커넥트 수수료 수익 (직접 실적)
또는
사용자가 다른 상품 수익 (간접 실적, 24시간 내)
    ↓
성과 기록 → Company Memory 학습
→ 다음 콘텐츠 성공률 개선
```

## 2. 네이버 쇼핑커넥트 수익 구조

### 직접 실적 (Direct Performance)

```
사용자의 쇼핑커넥트 링크 클릭
  ↓
"이 상품을 구매합니다" (구매 확정)
  ↓
네이버 쇼핑커넥트에서 판매자에게 주문 전달
  ↓
판매자가 판매 건수 기준으로 수수료 발생
  ↓
사용자가 쇼핑커넥트 대시보드에서 "직접 실적" 카운트 확인
  ↓
수수료: 상품별 2~3% 수준 (네이버 정책에 따라 변동)
```

**예시**:
- 제습제 8,900원 × 클릭 10명 × 1명 구매 × 2% = 178원
- 미니제습기 45,000원 × 클릭 20명 × 2명 구매 × 2% = 1,800원
- 월 평균: ₩500,000 목표 = 50~60개 상품 클릭 + 5~10% 구매율

### 간접 실적 (Indirect Performance)

```
사용자의 쇼핑커넥트 링크 클릭 후 24시간 내
  ↓
사용자가 다른 상품 구매 (사용자의 링크가 아닌 다른 상품)
  ↓
네이버 쇼핑커넥트에서 "간접 실적"으로 카운트
  ↓
수수료: 기본 1.8% (상품별 변동 가능)
```

**예시**:
- 링크 클릭 → 쇼핑백 추가 → 다른 상품 구매 → 간접 수익

### 수수료율 규칙

```
보장: 상품별로 다름
  - 일반 상품: 2.0%
  - 할인 상품: 1.8%
  - 캠페인 상품: 3.0% (기간 한정)

참고:
  - 반품 발생 시 수익 차감
  - 7일 집계 기한 (최종 확정)
  - 판매자 정산: 월 1회 (매월 20일경)
```

## 3. 수익 계획 (월별 목표)

### 초기 목표 (Month 1~3)

```
월 수익 목표: ₩500,000

필요한 조건:
- 콘텐츠: 주 2~3개 (주당 10~15개 링크 포함)
- 쇼핑 포함율: 80% (8~12개 글 중 10개가 쇼핑 포함)
- 클릭율: 평균 10% (100 조회 → 10 클릭)
- 구매율: 평균 3~5% (10 클릭 → 0.5 구매)

월 예상:
- 글 12개 × 평균 250 조회 = 3,000 조회
- 3,000 조회 × 10% 클릭율 = 300 클릭
- 300 클릭 × 4% 구매율 = 12 구매
- 평균 상품가: 20,000원
- 12 구매 × 20,000 × 2.5% 수수료 = ₩6,000 / 월?

→ 더 높은 조회수 or 더 비싼 상품 필요
```

### 성장 목표 (Month 4~12)

```
월 수익 목표: ₩1,000,000

전략:
1. 조회수 증가 (검색 최적화)
   - 블로그 초반 250 조회 → 6개월 후 600+ 조회
2. 쇼핑 포함율 개선
   - 80% → 90% (더 자연스러운 링크 배치)
3. 상품 선택 개선
   - 저가 상품 (5k~10k) 피함
   - 중가 상품 (20k~50k) 집중
   - 고가 상품 (50k+) 기회 활용
4. 간접 실적 활용
   - 직접: 간접 = 7:3 목표
```

## 4. 비용 구조

### Paperclip 자체 운영 비용 (사용자가 부담하지 않음)

```
개발·운영 비용 (초기):
  - 인프라: Vercel + Supabase = $0 ~ $50/월 (Free tier)
  - AI 모델: Claude / OpenAI = $100~500/월
  - 모니터링: Sentry 등 = $0~29/월
  - 도메인 + SSL = 자체 보유

유지보수 비용:
  - 개발자 1명 (현재)
  - 인프라 관리 (자동화)
  
→ 목표: 사용자에게 비용 청구 안 함 (오픈소스 또는 개인 프로젝트)
```

### 사용자의 쇼핑커넥트 비용

```
직접 비용: 없음
  (네이버가 수수료 차감, 사용자는 수익금만 받음)

기회 비용:
  - 게시 시간 (수동 게시, 자동화 안 함)
  - 이미지 편집 시간 (별도 도구 필요 가능)
  - 성과 기록 시간 (최소화 설계)

위험 비용:
  - 컴플라이언스 위반 시 계정 정지 위험
  - 낚시글로 신뢰도 하락
  - 자동 발행 정책 리스크 (자동 발행 금지)
```

## 5. 데이터베이스 설계 (수익 추적)

### 수익 관련 테이블

```
revenue_logs:
  id, product_id, amount (₩), revenue_type ("direct" / "indirect"),
  referrer_url, ordered_at, created_at

cost_logs:
  id, model ("claude" / "gpt-4"), task ("generate_blog" / ...),
  input_tokens, output_tokens, cost_usd, created_at

performance_logs:
  id, content_package_id, platform, views, clicks, likes,
  direct_revenue (₩), indirect_revenue (₩), recorded_at
```

### 수익 집계

```
월간 수익 요약:
  SELECT 
    DATE_TRUNC('month', ordered_at) as month,
    SUM(amount) as total_revenue,
    revenue_type,
    COUNT(*) as transaction_count
  FROM revenue_logs
  GROUP BY month, revenue_type;

글별 수익:
  SELECT 
    cp.topic_id,
    p.post_url,
    SUM(rl.amount) as total_revenue,
    pl.views,
    pl.clicks,
    (SUM(rl.amount) / pl.views * 100) as revenue_per_view_pct
  FROM content_packages cp
  JOIN performance_logs pl ON cp.id = pl.content_package_id
  LEFT JOIN revenue_logs rl ON ... (매핑 필요)
  GROUP BY cp.id, pl.id;
```

## 6. AI 비용 최적화

### 비용 구조

```
Claude API (기준):
  - Input: $0.003 / 1K tokens
  - Output: $0.015 / 1K tokens

OpenAI API:
  - GPT-4: $0.03 / 1K input, $0.06 / 1K output
  - GPT-3.5: $0.0005 / 1K input, $0.0015 / 1K output

예상 월 비용 (콘텐츠 10개 / 월):
  - 10개 글 × 5 프롬프트 호출 = 50회
  - 평균 input 2,000 tokens × 50 = 100,000 tokens
  - 평균 output 500 tokens × 50 = 25,000 tokens
  
  Claude: (100,000 * 0.003 + 25,000 * 0.015) / 1000 = $0.525
  
  → 월 $50 미만 비용으로 운영 가능 (저가 모델 선택 시)
```

### 비용 절감 전략

```
1. 응답 토큰 최소화
   - 정확한 프롬프트로 불필요한 설명 제거
   - 출력 형식 (JSON) 명확히 지정

2. 배치 처리
   - 하루 기회 3~5개를 1회 호출로 처리
   - 개별 호출보다 비용 효율

3. 캐시 활용
   - 자주 생성되는 패턴 캐시 (Category Playbook)
   - 이전 성공 패턴 템플릿 재사용

4. 모델 선택
   - 간단한 작업: Claude 3.5 Haiku (저가)
   - 복잡한 작업: Claude 3.5 Sonnet (중가)
   - 고급 작업: Claude 4 (고가, 필요 시만)
```

## 8. 위험 요소 및 대응

| 위험 | 영향 | 대응 |
|-----|------|------|
| **네이버 정책 변경** | 수수료율 하락, 링크 제한 | 다른 커머스 플랫폼 다각화 (쿠팡, 그마, 11번가) |
| **쇼핑커넥트 계정 정지** | 전체 수익 차단 | Compliance 엄격히, 자동 발행 금지 |
| **AI 비용 상승** | 월 200% 증가 가능 | 저가 모델 우선, 배치 처리, 캐시 |
| **낚시글 공격** | 조회 ↓, 신뢰도 ↓ | homefeed_score 낚시 감점, 제목-본문 일치 검증 |
| **경쟁 심화** | 같은 주제 포화 | Company Memory로 차별화된 각도 개발 |
| **사용자 이탈** | 매일 콘텐츠 만들다 지침 | 30분 최소화, 자동화 극대화, 성과 시각화 |

## 9. 성공 지표

### 단기 (Month 1)

```
- Paperclip HQ 접속률: 5일/주 이상
- 콘텐츠 생성: 주 1회 이상
- 블로그 게시: 모든 콘텐츠 발행 완료
- 성과 기록: 주 1회 이상
→ 목표: 시스템 신뢰성 입증
```

### 중기 (Month 3)

```
- 월 수익: ₩500,000 달성
- 평균 조회수: 250/글 이상
- 평균 클릭율: 8% 이상
- Company Memory: 성공 패턴 5개 이상 축적
→ 목표: 지속 가능성 입증
```

### 장기 (Month 12)

```
- 월 수익: ₩1,000,000 달성
- 평균 조회수: 500/글 이상
- 직접/간접 수익: 7:3 비율 달성
- 콘텐츠 자동화: 제작 시간 30분 이내 유지
→ 목표: 1인 AI 회사 운영의 현실성 입증
```

## 10. 확장 시나리오 (Phase 2+)

### SNS 채널 추가 (비용 vs 효과)

```
Phase 2: 네이버 클립 추가
  - 추가 조회: +30% 예상
  - 제작 시간: +5분 (대본 기반)
  - 비용: $0 (AI 활용)
  → 수익 효과: 직접 링크 불가, 블로그 재방문 유도

Phase 2: Instagram / Threads 추가
  - 추가 조회: +20% 예상
  - 제작 시간: +5분 (텍스트 변환)
  - 비용: $0
  → 수익 효과: 없음 (링크 규정 제한)
  → 가치: 브랜드 구축, 팔로워 확대

의사결정:
  - 클립: 필수 (수익 +30%)
  - SNS: 선택적 (브랜드용, 수익 간접적)
```

### 커머스 플랫폼 다각화

```
네이버 쇼핑커넥트: 현재 Primary
  - 수익: 월 ₩700,000 기대
  - 연계성: 높음 (블로그 → 검색 → 쇼핑)

쿠팡 파트너스 (추가 검토)
  - 수익: 2~4%
  - 진입: 상대적으로 쉬움
  - 복합도: 네이버와 다른 정책

그마/11번가 (향후)
  - 수익: 1~3%
  - 진입: 심사 필요
  - 효과: 추가 2~3%
```
