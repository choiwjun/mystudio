# 01-prd.md: 제품 요구사항 (PRD)

> Paperclip Company OS v0.7
> 기준일: 2026-07-02

---

## 문제 정리 (Problem Reframing)

### 처음 요청

"AI 글쓰기 도구가 필요하다"는 발상에서 출발했습니다.

### 다시 정리한 한 줄 문제

**혼자서 매일 수익형 콘텐츠를 기획 → 제작 → 검수 → 게시 → 회고까지 모두 돌리려면 회사 수준의 운영 구조가 필요한데, 그걸 사람 손으로 다 하면 지속이 안 된다.**

### 정리한 이유

- "AI 글쓰기만"으로는 부족함 — 블로그 글이 나왔어도 검색에 안 나오면, 쇼핑커넥트 링크가 없으면, 검수를 안 하면 소용없음
- 1인 회사가 네이버 홈피드·검색·클립·SNS를 모두 커버하려면 각 채널별 규칙(부서)이 있어야 함
- 매일 반복하려면 오너의 최종 판단만 필요하고, 나머지는 자동화된 의사결정(AI)이 미리 처리해야 함
- 그걸 실현하려면 "제품" 개발이 아니라 "운영 OS" 구축이 필요

## 1. 제품 정의

**Paperclip Company OS**는 1인이 매일 혼자 운영하지만, 내부적으로는 회사처럼 작동하는 AI 미디어커머스 운영 시스템입니다.

한 번의 승인으로 시장 조사 → 콘텐츠 기획 → 블로그 제작 → 쇼핑 수익화 → 클립/SNS 재활용 → 검수 → 성과 기록까지 모든 단계를 자동으로 진행합니다.

시스템 안에는:
- **Hermes** (시장조사팀): 시장 기회 발굴
- **Paperclip HQ** (경영실): 최종 판단과 승인
- **부서형 AI** (기획·제작·검수팀들): 각 단계 자동 실행
- **사용자** (오너): 최종 결정권자

라는 역할 분담이 있어서, 실제로는 회사처럼 돌아갑니다.

## 3. 하루 운영 흐름

```text
1. 사용자 Paperclip HQ 접속
2. Hermes가 오늘 기회 3~5개를 메모로 제시 (네이버 검색 API 기반 실데이터)
3. 각 기회의 홈피드/검색/수익/리스크 4축 점수 확인
4. 오늘 제작할 주제 1개 선택 (오너 승인)
5. HomeFeed Desk → 홈판형 제목, 썸네일, 첫 화면 구조 생성
6. Search Desk → 검색형 제목, 키워드 구조, FAQ, 비교표 생성
7. ShoppingConnect Desk → 상품별 추천 위치, CTA 설계
8. Naver Blog Desk → 블로그 원문 생성
9. Naver Clip Desk & Social Desk → 클립 대본, SNS 버전 생성 (Phase 2)
10. Compliance Desk → 광고표시, 가격 기준일, 출처, 과장 표현 검사
11. 통과하면 Export 가능 상태로 변환 (차단되면 수정 필요)
12. 사용자가 네이버 블로그 / 클립 / SNS에 수동 게시
13. 게시 URL, 조회수, 클릭, 수익 기록 (숫자 3~4개만)
14. Analytics → 성과를 Company Memory에 반영
15. Paperclip → 다음 주제 선택에 이 성과 데이터 활용
```

**총 시간**: 30분 이내 (입력은 최소화, AI 처리는 최대화)

## 4. 운영 원칙 11개

1. **블로그는 원본, 클립/SNS는 재활용** — 1개 주제당 블로그 1개를 만들고, 거기서 여러 채널용 변형본 생성
2. **네이버 블로그 자동 발행 금지** — API 종료 + 정책 리스크로 인해 수동 게시만 함
3. **쇼핑커넥트 링크 있으면 대가성 문구 필수** — 대가성 문구 없이는 Export 차단
4. **가격 있으면 가격 기준일 필수** — 언제 확인한 가격인지 명시
5. **직접 써보지 않은 상품은 후기체 금지** — 신뢰 리스크 + 법적 리스크
6. **홈판 제목은 강하게, 본문 첫 화면에서 약속 회수** — 낚시성 제목 방지
7. **링크 수보다 특정 상품을 직접 클릭할 이유 만들기** — 광고글 피로도 방지
8. **매일 성과 기록 + 잘 된 패턴만 반복** — Company Memory에 저장, 다음 기회에 활용
9. **고위험 카테고리 제외** — 건강, 투자, 법률, 의료, 다이어트 등은 초기 제외
10. **하루 최대 5개 기회 제시** — 정보 과잉 방지
11. **매일 콘텐츠 1개 기준** — 회사 구조만 만들고 실행 안 하는 리스크 방지

## 6. 리스크 및 대응

| 리스크 | 설명 | 대응 |
|-------|------|------|
| **홈피드 노출 보장 불가** | 네이버 홈피드는 세부 랭킹 로직 비공개 | 내부 "홈피드 적합도 점수"로 대리 최적화 |
| **낚시성 제목** | 클릭 후 이탈/숨김 → 알고리즘 역효과 | 낚시 위험을 homefeed_score 감점 요인으로 포함 |
| **광고글 피로도** | 링크 과다 배치 시 신뢰 하락 | 링크 수 제한, 문제 해결 우선 원칙 |
| **대가성 문구 누락** | 정책·신뢰 리스크 | Compliance Gate에서 Export 자동 차단 |
| **가격 정보 오래됨** | 상품형 글 신뢰도 하락, 재구매율 낮음 | price_checked_at 필드 필수 + 자동 갱신 알림 |
| **직접 사용 후기 위장** | 신뢰·법적 리스크 | 미사용 상품 후기체 금지 원칙 |
| **자동 발행 리스크** | 네이버 정책 위반 가능성 | 수동 게시 원칙 (Export 후 사용자가 직접) |
| **정보 과잉** | Hermes 후보가 너무 많으면 선택 지연 | 매일 최대 5개 Opportunity Memo만 제시 |
| **실행 지연** | 회사 구조만 만들고 게시 안 함 | 매일 콘텐츠 1개 기준 유지 |
| **쇼핑커넥트 상품 변경** | 상품 비활성화 / 수수료율 변경 | link_checked_at 필드 + 주기적 갱신 알림 |
| **네이버 검색 API 장애** | API 쿼터 초과 또는 응답 실패 | 폴백: "이전 스캔 결과 + AI 보완" 제시 |

## 7. 성공 기준

**Phase 1 완료 시점에 사용자가:**
- 매일 아침 HQ에 접속해서 기회 1개를 30분 안에 선택 및 게시 준비 완료
- 수동 게시 후 성과 3~4개 기록
- 2주 이상 지속 가능 (체계의 신뢰성 입증)
- 1개월 누적 5개 글의 성과 비교로 다음 선택 개선

이것이 "회사가 아닌 1인이 회사처럼 운영"하는 최소 증명입니다.
